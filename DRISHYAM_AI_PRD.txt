1.1 PRODUCT NAME
    Full Name   : DRISHYAM 1930
    Expanded    : Bharat Anti-Scam Intelligence Grid (BASIG)
    Short Name  : DRISHYAM

1.2 MISSION STATEMENT
    To protect every Indian citizen — from metro smartphone users to rural
    feature phone users — from financial scams by deploying real-time AI
    detection, agentic honeypot intelligence, fraud graph analysis, and
    post-scam recovery in one unified national platform.

1.3 ONE-LINE POSITIONING
    India's first end-to-end, real-time, AI-powered scam interception and
    citizen protection grid — built for 150 crore Indians.

1.4 PRODUCT VISION
    A world where no Indian picks up a scam call without being warned,
    no scammer operates without being profiled, and no victim suffers
    without receiving guided recovery support.

1.5 TARGET USERS
    PRIMARY CITIZENS
    - Indian citizens (smartphone + feature phone)
    - Senior citizens and low-literacy rural users
    - NRI / diaspora users
    - Women and vulnerable segment users

    INSTITUTIONAL PARTNERS
    - Telecom operators (Jio, Airtel, BSNL, Vi)
    - Banks (SBI, HDFC, ICICI, regional banks)
    - NPCI / UPI payment ecosystem
    - Government agencies (MHA, DoT, CERT-In, RBI, SEBI)
    - State cyber police units
    - NALSA / legal aid bodies
    - Insurance companies
    - Job platforms (Naukri, Indeed, LinkedIn India)

1.6 SCOPE BOUNDARIES
    IN SCOPE (v1)
    - Modules 1-15 as defined in this document
    - Hindi + English (MVP), 22 languages (full rollout)
    - Android + iOS mobile app
    - Web dashboard (agencies + citizens)
    - Feature phone layer (*1930# USSD + IVR + SMS)
    - India-only data storage and processing

    OUT OF SCOPE (v1)
    - Cross-border data sharing with foreign agencies (v2)
    - Hardware integration beyond existing mobile/telecom stack
    - Non-India language support

2.1 CURRENT SITUATION
    - India loses Rs. 7,000+ crore annually to phone and digital scams
    - 1930 cybercrime helpline receives millions of calls but lacks
      real-time interception capability
    - Most detection is reactive (post-scam complaint, not pre-scam warning)
    - Rural, elderly, and low-literacy citizens have zero protection tools
    - Scam intelligence is siloed across telecoms, banks, and police

2.2 PAIN POINTS BY USER TYPE

    CITIZENS
    - No warning before answering a suspicious call
    - No AI tool to handle the scammer while they stay safe
    - No guided help after a scam has occurred
    - No trusted dashboard to track their case status

    POLICE / AGENCIES
    - No consolidated fraud intelligence graph
    - Manual FIR preparation is slow and error-prone
    - No real-time surge alerts by district/state
    - Cross-agency data sharing is not automated

    TELECOMS / BANKS
    - No shared pre-transaction fraud signal
    - No automated mule account detection pipeline
    - No common threat intelligence feed

2.3 OPPORTUNITY
    DRISHYAM 1930 connects all these stakeholders on a single AI-powered
    platform — intercepting scams in real time, profiling scammers,
    protecting victims, and routing evidence to law enforcement automatically.

MODULE 1 — REAL-TIME TELECOM DETECTION GRID
Status: Built (core scoring, Redis cache, FRI API)

Purpose:
    Score every incoming call for fraud risk before it rings.

Key Features:
    - Fraud Risk Index (FRI) scoring: 0-100 in under 200ms
    - Input signals: SIM age, call velocity, CLI spoofed flag,
      prior complaints, geographic anomaly, SIM swap detection
    - Festival/ITR/salary-credit surge multiplier
    - Redis caching for repeat-number lookups
    - Routing rules: ALLOW / FLAG+WARN / ROUTE_TO_HONEYPOT
    - Scam weather forecast by district (predictive surge)
    - Tower mesh alert trigger for geographic burst events
    - Operator dashboard for flagged calls

Acceptance Criteria:
    [AC-M1-01] FRI score computed in < 200ms
    [AC-M1-02] Normal 3-year SIM scores < 30
    [AC-M1-03] New SIM + 200 calls/day scores > 85
    [AC-M1-04] CLI spoofed flag adds >= 20 to base score
    [AC-M1-05] Score never exceeds 100 or goes below 0
    [AC-M1-06] Score > 85 triggers ROUTE_TO_HONEYPOT action
    [AC-M1-07] FRI logged with timestamp in PostgreSQL
    [AC-M1-08] Dashboard reflects score within 5 seconds

    - Psychological exploitation index scoring
    - Background audio fingerprinting (call centre pod detection)

    - Safe call termination rules
    - Transcript storage and full-text indexing

Acceptance Criteria:

    - Recovery Companion prompt appears in app
